Understand npm Basics

npm (Node Package Manager) lets you install JavaScript libraries. Here are the commands you'll use:

# Install all packages listed in package.json
npm install

# Add a new package to your project
npm install package-name

# Run development server
npm run dev

# Build for production
npm run build
üìÅ WHAT'S package.json?

It's a file that lists all the packages your project needs. When you run npm install, it reads this file and downloads everything.

Understanding node_modules

When you run npm install, it creates a node_modules folder with all downloaded packages.

  • This folder can be HUGE (hundreds of MB)
  • Never commit it to Git (it's in .gitignore by default)
  • If you delete it, just run npm install again
  • Each project has its own node_modules

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!

Fetch Data from Supabase

// In any page or component:

import { supabase } from '@/lib/supabase'

async function getListings() {
  const { data, error } = await supabase
    .from('listings')
    .select('*')
    .order('created_at', { ascending: false })
  
  if (error) {
    console.error('Error:', error)
    return []
  }
  
  return data
}

// Insert data:
const { data, error } = await supabase
  .from('listings')
  .insert({
    name: 'ABC Cleaning',
    price: 250000,
    location: 'Denver, CO'
  })

Enable Row Level Security (RLS)

RLS controls who can read/write data. For public read access:

  • Go to Authentication ‚Üí Policies
  • Select your table
  • Click "New Policy"
  • Choose "Enable read access for all users"
  • Click "Review" ‚Üí "Save policy"
üìñ RLS EXPLAINED

Without RLS policies, no one can read your data (even with the anon key). You must explicitly allow access. Start with "read access for all" for public data.

Your First Script - Connect to Supabase

# Create file: scraper.py

import os
from dotenv import load_dotenv
from supabase import create_client

# Load environment variables
load_dotenv()

# Connect to Supabase
supabase = create_client(
    os.getenv("SUPABASE_URL"),
    os.getenv("SUPABASE_SERVICE_KEY")
)

# Fetch all listings
response = supabase.table('listings').select('*').execute()
print(f"Found {len(response.data)} listings")

# Insert a new listing
new_listing = {
    "name": "Test Cleaning Business",
    "price": 150000,
    "location": "Denver, CO"
}
result = supabase.table('listings').insert(new_listing).execute()
print("Inserted:", result.data)

Understand DNS Record Types

A Record Points domain to an IP address
CNAME Points domain to another domain
MX Mail server (for email)
TXT Text records (verification, security)

Connect Domain to Vercel

Step 1: In Vercel, add your domain (Settings ‚Üí Domains)

Step 2: Vercel shows you what records to add. Usually:

Type: A Name: @ | Value: 76.76.21.21
Type: CNAME Name: www | Value: cname.vercel-dns.com

Step 3: Add these records at your registrar:

  • Log in to Name.com (or your registrar)
  • Find your domain ‚Üí DNS Records / Manage DNS
  • Add new record:
# A Record for root domain
Type: A
Host/Name: @
Value/Points to: 76.76.21.21
TTL: 3600 (or Auto)

# CNAME for www subdomain
Type: CNAME
Host/Name: www
Value/Points to: cname.vercel-dns.com
TTL: 3600

Wait for DNS Propagation

DNS changes take time to spread globally (5 minutes to 48 hours, usually under 30 mins).

  • Check status at whatsmydns.net
  • Enter your domain and select "A" record
  • Green checkmarks mean it's propagated to that location

Verify in Vercel

  • Go back to Vercel ‚Üí Settings ‚Üí Domains
  • Your domain should show "Valid Configuration"
  • Vercel automatically creates SSL certificate (HTTPS)
  • Visit your domain - should show your site!

Add MX Records for Email

MX records tell email servers where to send mail for your domain:

# Add ALL of these MX records

Priority: 1
Host: @
Value: ASPMX.L.GOOGLE.COM

Priority: 5
Host: @
Value: ALT1.ASPMX.L.GOOGLE.COM

Priority: 5
Host: @
Value: ALT2.ASPMX.L.GOOGLE.COM

Priority: 10
Host: @
Value: ALT3.ASPMX.L.GOOGLE.COM

Priority: 10
Host: @
Value: ALT4.ASPMX.L.GOOGLE.COM
‚ö†Ô∏è DELETE OLD MX RECORDS

If you have existing MX records from your registrar, delete them first. Only Google's MX records should exist.

Add SPF Record (Prevent Spam Flags)

SPF tells receiving servers that Google is authorized to send email for your domain:

Type: TXT
Host/Name: @
Value: v=spf1 include:_spf.google.com ~all
TTL: 3600
