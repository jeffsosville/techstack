‚ö° Supabase

Database and backend - stores all your data

üåê Supabase.com üìö Docs

What Supabase Provides

  • PostgreSQL - Full SQL database
  • Auth - User signup/login
  • Storage - File uploads
  • Real-time - Live data updates
  • API - Auto-generated REST

Key Concepts

  • Tables - Like Excel sheets
  • RLS - Row Level Security
  • Foreign Keys - Link tables
  • anon key - Public, respects RLS
  • service key - Bypasses RLS (server only!)

Create Account & Project

  • Go to supabase.com
  • Sign up with GitHub
  • Click "New Project"
  • Name it, create strong password, select region
  • SAVE THE PASSWORD!

Get API Keys

  • Settings (gear) ‚Üí API
  • Copy Project URL and anon public key

Add to your .env.local:

NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...

Create a Table

  • Click Table Editor
  • Click "Create a new table"
  • Name: "listings"
  • Add columns: name (text), price (int4), location (text)
  • Click Save

Install in Next.js

npm install @supabase/supabase-js @supabase/ssr

Create client file:

// lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)

Fetch Data

import { supabase } from '@/lib/supabase'

const { data, error } = await supabase
  .from('listings')
  .select('*')

Enable RLS Policy

  • Authentication ‚Üí Policies
  • Select your table
  • New Policy ‚Üí "Enable read access for all users"
  • Save
üìñ RLS

Without policies, no one can read data. Add "read access for all" for public data.

üåê Domain & DNS

Connect your domain to hosting

üåê Name.com üîç Check DNS

DNS Record Types

  • A Record - Points to IP address
  • CNAME - Points to another domain
  • MX - Mail server
  • TXT - Verification records

How It Works

  • User types your domain
  • DNS looks up IP address
  • Browser connects to Vercel
  • Your site loads!

Buy a Domain

  • Name.com - Simple, good prices
  • Namecheap - Budget-friendly
  • Cloudflare - At-cost pricing

Search, add to cart, checkout. ~$10-15/year for .com

Add Domain in Vercel

  • Vercel ‚Üí Project ‚Üí Settings ‚Üí Domains
  • Enter your domain (e.g., cleaningexits.com)
  • Vercel shows records to add

Add DNS Records

At your registrar (Name.com), add:

# A Record for root domain
Type: A
Name: @
Value: 76.76.21.21

# CNAME for www
Type: CNAME
Name: www
Value: cname.vercel-dns.com

Wait for Propagation

  • DNS changes take 5 min to 48 hours
  • Usually under 30 minutes
  • Check at whatsmydns.net

Verify in Vercel

  • Vercel ‚Üí Domains should show "Valid"
  • SSL certificate auto-created
  • Visit your domain!

üìß Google Workspace

Professional email with your domain

üåê Google Workspace ‚öôÔ∏è Admin Console

What You Get

  • Custom Email - you@yourdomain.com
  • Gmail Interface - Same Gmail
  • Google Drive - 30GB per user
  • Admin Console - Manage users

DNS Records Needed

  • TXT - Domain verification
  • MX - Mail servers (5 records)
  • TXT - SPF (prevent spam)
  • TXT - DKIM (authentication)

Sign Up

  • Go to workspace.google.com
  • Click "Get Started"
  • Enter business name, your domain
  • Create admin account (e.g., jeff@yourdomain.com)

Verify Domain

Google gives you a TXT record to add:

Type: TXT
Name: @
Value: google-site-verification=xxxx

Add at your registrar, click Verify in Google.

Add MX Records

Delete old MX records, add all 5 Google MX records:

Priority 1:  ASPMX.L.GOOGLE.COM
Priority 5:  ALT1.ASPMX.L.GOOGLE.COM
Priority 5:  ALT2.ASPMX.L.GOOGLE.COM
Priority 10: ALT3.ASPMX.L.GOOGLE.COM
Priority 10: ALT4.ASPMX.L.GOOGLE.COM

Add SPF Record

Prevents your emails from going to spam:

Type: TXT
Name: @
Value: v=spf1 include:_spf.google.com ~all

Set Up DKIM

  • Admin Console ‚Üí Apps ‚Üí Gmail ‚Üí Authenticate email
  • Generate new record
  • Add TXT record at registrar:
Type: TXT
Name: google._domainkey
Value: [long string from Google]

Add Users & Aliases

  • Admin Console ‚Üí Users ‚Üí Add user
  • Or add aliases to one inbox:
  • support@, info@, sales@ ‚Üí all go to jeff@

Access Email

  • Go to gmail.com
  • Sign in with jeff@yourdomain.com
  • Done!
