‚ö° Supabase

Your database and backend-as-a-service. Think of it as the "brain" that stores all your data and handles user authentication.

What Supabase Provides

  • PostgreSQL Database - Full SQL database, not NoSQL
  • Authentication - User signup, login, OAuth (Google, etc.)
  • Storage - File uploads (images, PDFs)
  • Real-time - Live data updates via websockets
  • Edge Functions - Serverless functions (Deno)
  • Auto-generated API - REST & GraphQL from your schema

Key Concepts

  • Tables - Like Excel sheets, store your data
  • Row Level Security (RLS) - Control who sees what
  • Foreign Keys - Link tables together
  • Views - Pre-built queries for common data
  • Functions - Stored procedures in PostgreSQL

  .select('*')

üí° Server vs Client Components

By default in App Router, components are Server Components (run on server, no interactivity). Add "use client" at the top of a file to make it a Client Component (runs in browser, can use useState, onClick, etc.).

# Example: Scrape and store in Supabase
import requests
from bs4 import BeautifulSoup
from supabase import create_client
import os
from dotenv import load_dotenv

load_dotenv()  # Load .env file

# Connect to Supabase
supabase = create_client(
    os.getenv("SUPABASE_URL"),
    os.getenv("SUPABASE_KEY")
)

# Scrape a page
response = requests.get("https://example.com/listings")
soup = BeautifulSoup(response.text, 'html.parser')

# Extract data
listings = []
for item in soup.select('.listing-card'):
    listings.append({
        "name": item.select_one('.title').text,
        "price": item.select_one('.price').text,
        "url": item.get('href')
    })

# Bulk insert to Supabase
supabase.table('listings').upsert(listings).execute()

üí° Service Key vs Anon Key

Anon Key: Safe for browser/frontend, respects Row Level Security (RLS).
Service Key: Bypasses RLS, use only in server-side Python scripts. Never expose in frontend!

üí° SPF, DKIM, DMARC

These three DNS records prevent your emails from going to spam. SPF says who can send email for your domain. DKIM cryptographically signs emails. DMARC tells receivers what to do if checks fail. Google Workspace guides you through setup.
